Final Year Thesis Project (E) 448

Advanced IoT Health Tracking System with Real-Time Data Display

This project delivers a cost-effective IoT health monitor that measures heart rate, oxygen saturation (SpO2), and temperature, then publishes real-time and historical data through a web dashboard and an external TFT display.

Scope note: this system is designed for personal monitoring support and trend tracking, not clinical diagnosis.

Project goals and system scope

Problem addressed

Existing monitoring products can be expensive and inaccessible. The thesis targeted an affordable architecture that still provides structured data, trend visibility, and remote access.

Core objectives

  • Measure heart rate, SpO2, and temperature reliably.
  • Show data in real time on both device display and web server.
  • Store and visualize historical data for trend analysis.
  • Add alerting for abnormal readings and protect user data.

Hardware design

Sensing layer

  • SEN-15219 for heart rate and SpO2 measurement.
  • D6T-1A-01 infrared sensor for non-contact temperature.
  • Temperature calibration offset used: 3.34 C.

Processing and connectivity

  • ESP32-C3 selected for built-in Wi-Fi and low cost.
  • I2C used for sensor bus integration.
  • Station mode networking enables browser-based monitoring.

Power and interface

  • 9V input stepped down to 5V through LM7805CV regulator.
  • 3.3V rail from ESP32 powers the SEN-15219 module.
  • Adafruit LLC 2050 TFT display connected via SPI (24 MHz).

Physical implementation

  • Custom PCB and enclosure designed for compact integration.
  • Full schematic, PCB layers, and CAD models documented in appendices.
  • Final build tested for signal integrity and reliability.

Software design

Embedded workflow

  • Sensor initialization and bus setup in device startup sequence.
  • Continuous acquisition loop for heart rate, SpO2, and temperature.
  • Input validation and abnormal measurement handling.

Dashboard and data flow

  • Local web server on ESP32 for dashboard, detail, and info pages.
  • Chart.js used to display dynamic trends and comparisons.
  • Data pushed to Firebase for historical mode and averages.

Security and persistence

  • User registration and login flow on the web interface.
  • Credentials persisted with non-volatile storage (NVS).
  • Historical data retained in cloud storage for long-term access.

User alerting

  • Web and LCD alert behavior for out-of-range values.
  • Dashboard refresh cadence: every 1 second.
  • Color and notification cues for normal vs abnormal states.

Future expansion paths

  • Add additional metrics such as BP, ECG, and respiratory rate.
  • Move to battery-powered operation for portability.
  • Integrate with healthcare records and provider workflows.
  • Reduce size toward a compact wearable form factor.
